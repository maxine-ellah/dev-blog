Problem solving

14 February 2016

Tell your non-tech friend a story about a time you 1. Got blocked on a simple problem 2. Solved a problem in an elegant way In both cases:

What was the problem?

My very first step to making the racer game was to figure out how to move an 'active' class from one tag to another using javascript.

What problem solving techniques did you use?

I googled so damn much and got some great tips on stackoverflow. For this particular problem I had to get a better understanding of for loops for counting through the tags and selectors to target the right elements. I asked alot of really basic questions on stackoverflow to understand these things as thoroghly as I could.


How did you feel throughout the process?

Really insecure to be perfectly honest. I know I'm not supposed to know everything about javascript but I really struggled with the fact that I wasn't taking in all the information that I had previously learned about javascript and had to rely alot on other people's advice and code. However I think this process really made me look at heaps of code and (kind of) understand the core of common statements.

.map()

This applies a function to multiple elements in an array and then puts the outcomes into a new array.

.filter()

This literally filters the contents of the array through a funtion to produce a new list (array) of values which are true in the function e.g. .filter() with % 5 == 0 would test all the numbers in an array and all the ones divisible by 5 would be put in the new array.

.reduce()

As far as I can understand this applies a function to (or adds up) the contents of the array from left to right, so that one value is left.
